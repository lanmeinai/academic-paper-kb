# Skill：处理新论文

## 触发方式
用户说「处理新论文」或「处理 inbox」时执行本 Skill。

## 执行步骤

### 步骤 1：扫描 inbox/
列出 inbox/ 目录下所有 .pdf 文件，跳过已在 data/index.json 中记录的文件。

### 步骤 2：逐篇提取（调用 extract_pdf.py）
对每个新 PDF 运行：
```bash
python skills/处理新论文/extract_pdf.py "<pdf路径>"
```
提取结果写入临时 JSON。

### 步骤 3：调用 DeepSeek API 分析
调用 skills/处理新论文/analyze_paper.py，传入提取的文本，让 DeepSeek 返回：
- 中文标题
- 中文摘要（200字以内）
- 核心贡献（一句话）
- 方法概述（3-5句话）
- 方法标签（根据论文内容自动生成，见标签体系）
- 自由标签（1-3个补充标签）
- 实验数据摘录（数据集/样本名称、核心指标数值）
- 参考文献列表（标题列表，用于后续匹配库内论文）

### 步骤 4：更新 data/index.json
将分析结果追加写入索引。

### 步骤 5：移动 PDF
按主标签将 PDF 移入 papers/<分类>/ 目录。
文件名标准化为：{年份}_{第一作者姓}_{英文标题前30字符}.pdf

### 步骤 6：生成 Obsidian 笔记
调用 skills/处理新论文/build_note.py，生成 vault/论文笔记/<文件名>.md

### 步骤 7：更新主题索引页和作者索引页

### 完成后输出处理摘要
列出：处理了哪些文件、打了哪些标签、建立了哪些引用链接。

## 标签体系

### 标签生成原则
AI 会根据论文内容自动生成标签，维度包括：

### 方法类
论文使用的方法论特征（如：实验研究、理论分析、数值模拟、案例分析、实证研究、元分析、综述等）

### 技术类
论文使用的具体技术（如：深度学习、统计分析、优化算法、仿真模拟等，根据论文实际内容确定）

### 领域类
论文所属的研究领域和子领域（根据论文实际内容自动识别）

### 任务类
论文要解决的具体任务类型（根据论文实际内容自动识别）

### 自由标签
以上之外，根据论文特有内容自由添加 1-3 个补充标签。
