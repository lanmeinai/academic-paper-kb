# 📚 学术论文知识库

> 把 PDF 扔进 inbox，剩下的交给 Claude Code。

---

## 这个项目做什么

你把收集到的学术论文 PDF 放入 `inbox/` 文件夹，然后告诉 Claude Code「处理新论文」，它会自动完成：

1. **提取论文内容** — 用 PyMuPDF 读取 PDF 全文
2. **AI 深度分析** — 调用 DeepSeek API，提取中文标题、摘要、核心贡献、实验数据、参考文献列表
3. **自动打标签分类** — AI 根据论文内容自动生成多维标签（方法类/技术类/领域类/任务类）
4. **移动并归档 PDF** — 按主标签移入 `papers/<分类>/` 子目录
5. **生成 Obsidian 笔记** — 中文内容，含实验数据表格、引用关系 WikiLink
6. **建立引用关系网络** — 自动识别库内论文之间的引用关系，双向链接

---

## 特点

- 🌐 **适配任何学术领域** — 不限于特定学科，AI 自动识别论文领域并生成标签
- ⚙️ **可配置领域偏好** — 在 `config.json` 中设置研究方向，AI 分析更精准
- 📝 **全中文笔记** — 英文论文自动生成中文摘要和标签
- 🔗 **引用关系图谱** — 自动建立论文间的引用网络，Obsidian 中可视化浏览

## 目录结构

```
inbox/              ← 📥 把新 PDF 放这里
papers/
  ├── 实验研究/     ← 处理完的 PDF 自动归档到这里（按主标签分类）
  ├── 深度学习/
  └── ...
data/
  └── index.json    ← 所有论文的结构化索引
vault/              ← 用 Obsidian 打开这个目录
  ├── 论文笔记/     ← 每篇论文一个中文笔记
  ├── 主题索引/     ← 按标签聚合的论文列表
  └── 作者索引/     ← 按作者聚合的论文列表
skills/             ← Claude Code 的技能文件
config.json         ← 用户研究领域配置
exports/            ← 导出的参考文献文件
```

## 使用方法

### 第一次使用

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 DeepSeek API Key
cp .env.example .env
# 编辑 .env，填入你的 Key

# 3. （可选）配置你的研究领域
# 编辑 config.json，设置 research_domain 和 domain_hint

# 4. 用 Obsidian 打开 vault/ 目录
# 推荐安装插件：Dataview、Graph Analysis
```

### 日常使用流程

```
第一步：把新下载的论文 PDF 复制到 inbox/ 文件夹

第二步：打开 Claude Code，进入项目目录，说：
        「处理新论文」

第三步：等待 Claude Code 处理完成（每篇约 30-60 秒）

第四步：打开 Obsidian，在关系图谱中查看新增论文的位置
```

## 三个核心功能

### 📥 功能一：处理新论文
对 Claude Code 说：**「处理新论文」**

Claude Code 会处理 inbox/ 中所有未处理的 PDF，完成后输出摘要：
- 处理了哪些文件
- 打了哪些标签
- 建立了哪些引用链接

---

### ✍️ 功能二：写论文辅助
对 Claude Code 说：**「写论文辅助」** 或 **「帮我写 Related Work」**

Claude Code 会询问你的论文主题，然后输出：
- Related Work 推荐章节结构（3-5个子章节）
- 每个章节推荐引用的库内论文（说明引用理由）
- 必引核心论文清单
- 研究空白分析（你的论文切入点）
- 该方向技术演进时间线

---

### 📚 功能三：导出参考文献
对 Claude Code 说：**「导出参考文献」**

Claude Code 会询问筛选条件，然后生成 BibTeX 文件，保存到 `exports/` 目录。

## 领域适配指南

### 设置研究领域

编辑 `config.json`：

```json
{
  "research_domain": "计算机视觉",
  "domain_hint": "重点关注目标检测、图像分割和生成模型方向的论文"
}
```

### 标签体系

标签由 AI 根据论文内容自动生成，分为四个维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| 方法类 | 方法论特征 | 实验研究、理论分析、数值模拟、综述 |
| 技术类 | 具体技术 | 深度学习、统计分析、优化算法 |
| 领域类 | 研究领域 | 计算机视觉、自然语言处理、材料科学 |
| 任务类 | 目标任务 | 目标检测、文本分类、性能优化 |

## Obsidian 使用技巧

### 关系图谱
- 打开左侧「关系图谱」，每个节点是一篇论文
- 节点名称格式：`{年份}_{作者}_{标题关键词}`
- 连线代表引用关系，越粗说明被引用越多
- 颜色按标签区分研究方向

### Dataview 查询示例
在任意笔记中插入以下代码块（需安装 Dataview 插件）：

```dataview
TABLE 年份, 标签, 核心贡献
FROM "论文笔记"
WHERE contains(标签, "深度学习")
SORT 年份 DESC
```

## 常见问题

**Q：PDF 是扫描版，没有可提取的文字怎么办？**  
扫描版 PDF 无法提取文本，Claude Code 会报告该文件并跳过。建议用 OCR 工具（如 Adobe Acrobat）预处理后再放入 inbox/。

**Q：DeepSeek 分析结果不准确怎么办？**  
打开对应的 vault/论文笔记/ 中的 .md 文件，手动修改任何字段，Obsidian 的关系图谱会实时更新。

**Q：想重新处理某篇已处理的论文？**  
从 data/index.json 中删除该论文的记录，将 PDF 重新放入 inbox/，再次运行「处理新论文」。

**Q：引用关系识别不全怎么办？**  
在笔记的「引用了（库内论文）」部分手动添加 `[[笔记文件名]]` 即可，Obsidian 会自动建立双向关系。

**Q：如何切换到不同的研究领域？**  
编辑 `config.json` 中的 `research_domain` 即可。已处理的论文不受影响，新处理的论文会使用新的领域设定。
